# Isinstance Function 
আপনার দেয়া উদাহরণগুলো পাইথনের isinstance() ফাংশনের সঠিক ব্যবহার প্রদর্শন করছে। এই ফাংশনটি মূলত কোনো একটি অবজেক্ট নির্দিষ্ট কোনো ডাটা টাইপ বা ক্লাসের অন্তর্ভুক্ত কি না, তা যাচাই করার জন্য ব্যবহৃত হয়।

এর গাণিতিক বা লজিক্যাল গঠনটি অনেকটা এরকম: isinstance(object, classinfo)

এএখানে আপনার কোডগুলোর ব্যাখ্যা নিচে দেওয়া হলো:

isinstance('Hello world', str): এখানে অবজেক্টটি একটি স্ট্রিং, তাই এটি True রিটার্ন করে।

isinstance(True, bool): পাইথনে True এবং False হলো বুলিয়ান টাইপ, তাই এটি True।

isinstance(42, int): ৪২ একটি পূর্ণসংখ্যা (Integer), তাই ফলাফল True।

isinstance('John Doe', int): এখানে 'John Doe' একটি স্ট্রিং, কিন্তু যাচাই করা হচ্ছে এটি ইন্টিজার কি না। যেহেতু টাইপ মিলছে না, তাই এটি False রিটার্ন করছে।

একটি মজার তথ্য:
পাইথনে bool ক্লাসটি আসলে int ক্লাসের একটি সাব-ক্লাস। তাই আপনি যদি isinstance(True, int) চেক করেন, তবে সেটিও কিন্তু True আসবে!



## Integer Vs Float 

১. ইন্টিজার (Integer): এটি যেকোনো পূর্ণসংখ্যাকে বোঝায়। এতে কোনো দশমিক থাকে না (যেমন: ১০, -৫, ৪২)। এটি মেমরিতে জায়গা কম নেয় এবং গণনার ক্ষেত্রে সবসময় একদম সঠিক মান প্রদান করে।

২. ফ্লোট (Float): এটি দশমিক যুক্ত সংখ্যাকে বোঝায় (যেমন: ১০.৫, -৫.২, ৩.০)। দশমিকের পরের সূক্ষ্ম হিসাব রাখার জন্য এটি ইন্টিজারের তুলনায় বেশি মেমরি ব্যবহার করে। তবে বড় বা জটিল গাণিতিক হিসাবের ক্ষেত্রে অনেক সময় দশমিকের একদম শেষের দিকে সামান্য পার্থক্য (Rounding error) হতে পারে।

সংক্ষেপে মনে রাখার উপায়:

Integer = পূর্ণসংখ্যা (গাড়ি কয়টি, মানুষ কয়জন)।

Float = ভগ্নাংশ বা দশমিক (তাপমাত্রা কত, জিনিসের দাম কত)।
